## 背景
项目目前没有版本控制，需要引入 Git 来管理代码变更历史和支持团队协作。

## 目标 / 非目标
- 目标：建立标准的 Git 版本控制流程
- 目标：确保敏感数据不被提交到版本控制
- 非目标：配置远程仓库（留待后续）
- 非目标：设置 CI/CD 流程

## 决策
- 决策：使用 Git 作为版本控制系统
  - 原因：行业标准、工具生态完善、支持分布式开发
- 替代方案：SVN、Mercurial
  - 理由：Git 更现代、社区更大、学习资源更多

## 风险 / 权衡
- 本地开发数据可能被误提交 → 通过 .gitignore 保护
- 初期学习成本 → 通过清晰的文档降低

## 实施计划
1. 初始化仓库
2. 配置忽略规则
3. 添加项目文件
4. 创建初始提交
